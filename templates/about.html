Goal

This web application will use a combination of various industry statistics in order to optimize air travel within Europe. Goals of this project come from both an economic and environmental standpoint. By optimizing flight paths given a set of parameters from the users, this application will be able to find the most efficient way to travel in terms of aircrafts used, layovers, etc. Efficiency inherently lowers overall cost for an airline and even ticket price for travelers. Fuel usage is also linked with environmental impact of an airline, since increased efficiency minimizes unnecessary fuel consumption. Our combined goal is to bring these cost saving efficiencies to the foreground of aviation, and make their use as accessible as possible to the both new and old airlines interested in increased optimization.

Users

The landing page of the website presents the user with two options: "Try" and "About." Respectively identical options are available in the header as "Generator" and "About." Upon selecting the about option, details about the applications design, and motivations are accessible. If the user clicks the try or generator options, they are brought to a new page, "Airfleet Generator." This page has two drop down menus, allowing the user to pick desired departure and arrival airports for their flight. Once an airport is picked in one, it is no longer an option in the other. Chosen routes are assumed to be symmetric, that is to say that if a user wants to fly one direction, they will inevitably need to make a return trip to the start location. Next, the user chooses the number of passengers they expect to fly over the selected route. The "Show Result" button is non-active until every metric has been filled with valid responses. After selecting "Show Result," a table is displayed showing details on an optimized flight plan and a total cost of the given route.
